# Interiority Semantic Change Analysis

Computational analysis of semantic change in Chinese "interiority" words (心里, 心中, 内心, etc.) across five historical periods using temporal referencing Word2Vec (TempRef).

## Periods


| Period       | Label          | Approximate Dates |
| ------------ | -------------- | ----------------- |
| Ming-Qing    | `mingqing`     | 1368 -- 1860      |
| Late Qing    | `late_qing`    | 1860 -- 1911      |
| Republican   | `republican`   | 1911 -- 1949      |
| Socialist    | `socialist`    | 1949 -- 1976      |
| Contemporary | `contemporary` | 1976 -- present   |

## CLI Workflow

All commands are run from the project root.

### Step 1: Normalize texts

Convert traditional Chinese to simplified, remove noise, normalize punctuation and filenames.

```bash
# Normalize all periods at once (data/texts/* → data/texts_normalized/*)
python scripts/normalize_texts.py

# Or normalize a single period / file
python scripts/normalize_texts.py data/texts/mingqing --output-dir data/texts_normalized/mingqing
python scripts/normalize_texts.py input.txt --output-dir custom_output/
```

### Step 2: Segment texts

Split normalized texts into words using jieba (enhanced with vocab from pre-modern Chinese dictionary) and group them by period. Uses multiprocessing for faster segmentation. This produces per-period `.txt` files in `data/segmented/` (one sentence per line, space-separated words).

```bash
python scripts/segment_texts.py

# With custom worker count (default: 4)
python scripts/segment_texts.py --workers 2

# Full options
python scripts/segment_texts.py --input-dir data/texts_normalized --output-dir data/segmented --workers 4 --min-length 5
```

### Step 3: Train the TempRef model

Train a TempRefWord2Vec model on the segmented data from Step 2. Target words from `data/dictionaries/interiority_words.txt` are replaced with a unified token (`interiority` by default). Model hyperparameters are defined at the top of `scripts/train_tempref.py` and can be overridden via CLI flags.

```bash
python scripts/train_tempref.py

# Custom data directory and output
python scripts/train_tempref.py --data-dir data/segmented --output models/tempref_interiority_w2v.npy

# Override defaults:
python scripts/train_tempref.py --epochs 5 --vector-size 300 --replacement another_word
```

### Step 4: Analyze semantic changes

Run semantic change analysis using the model from Step 3 and the segmented data from Step 2, and output results as CSV files.

```bash
python scripts/semantic_change.py

# Custom paths
python scripts/semantic_change.py --model models/tempref_interiority_w2v.npy --data-dir data/segmented --output-dir results/
```

This produces:

- One CSV per transition (e.g., `results/semantic_changes_mingqing_to_late_qing.csv`)
- A summary CSV of temporal variant similarities (`results/temporal_variant_similarities.csv`)
- Plots in `images/`

## Statistical Stability and Significance Testing

The `semantic_change_multiseed.py` script estimates result stability and statistical significance:

1. **Multi-seed stability**: Trains multiple times with different random seeds to measure variance from Word2Vec's stochastic training.
2. **Permutation test**: Shuffles sentences across period labels to build a null distribution of change scores.

### Quick Start

```bash
nohup ./scripts/run_multiseed_analysis.sh > multiseed_analysis.log 2>&1 &

# Monitor progress: tail -f multiseed_analysis.log
# Stop the process: pkill -f semantic_change_multiseed
```

Or run steps manually:

```bash
# Step 1: Build null distribution
python scripts/semantic_change_multiseed.py --trials 100 --permute-periods --output-dir results/null_distribution

# Step 2: Run multi-seed analysis with p-values
python scripts/semantic_change_multiseed.py --trials 100 --output-dir results/multiseed --null-dir results/null_distribution
```

### Output

- `multiseed_scores.npz` / `null_scores.npz`: Raw scores per word per trial
- `semantic_changes_{transition}.csv`: Filtered results sorted by mean_change

**Multi-seed CSV columns:**


| Column                    | Description                            |
| ------------------------- | -------------------------------------- |
| `word`                    | Reference word                         |
| `mean_change`             | Mean change score across trials        |
| `std_change`              | Standard deviation across trials       |
| `ci_low`, `ci_high`       | 2.5th and 97.5th percentiles           |
| `n_trials`                | Number of trials                       |
| `z_score`                 | Signal-to-noise ratio (mean / std)     |
| `count_pre`, `count_post` | Word frequency in source/target period |
| `cooc_pre`, `cooc_post`   | Co-occurrence with interiority words   |
| `null_mean`               | Mean change from null distribution     |
| `p_value`                 | Fraction of null scores ≥ observed     |


**Null CSV columns:** `word`, `mean_change`, `std_change`, `ci_low`, `ci_high`, `n_trials`, `z_score`

### Key Options


| Option                  | Default                     | Description                                  |
| ----------------------- | --------------------------- | -------------------------------------------- |
| `--trials`              | 100                         | Number of training runs                      |
| `--permute-periods`     | off                         | Shuffle sentences across periods (null mode) |
| `--null-dir`            | `results/null_distribution` | Path to null distribution for p-values       |
| `--checkpoint-interval` | 10                          | Save progress every N trials                 |
| `--resume`              | off                         | Resume from existing checkpoint              |
| `--min-count`           | 5                           | Min word count in each period                |
| `--min-cooc`            | 5                           | Min co-occurrence with target words          |
| `--postag`              | `n.`*                       | POS filter regex (`none` to disable)         |


### Interpretation

- **High z-score (>3)**: Result is consistent across training runs
- **Low p-value (<0.05)**: Result exceeds what's expected under null hypothesis
- Words with both high z-score and low p-value are strongest candidates for genuine semantic change
